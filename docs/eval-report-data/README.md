## Data Dictionary

| Column | Type | Description |

### `results.json` Row Structure

Each entry contains the CSV fields plus nested objects:

- `tokens.{ input, output, reasoning, cacheRead, cacheWrite, total, active }`
- `timing.{ wallMs, segments[] }`
- `toolCalls.{ total, byCategory, failed, retried, errorRate, records[] }` -- `records` contains per-call detail: tool name, category, success, duration
- `cost.{ totalUsd, inputUsd, outputUsd, reasoningUsd }`
- `checkpointDetails[]` -- per-checkpoint id, description, passed/failed, and the actual GitHub API response used for verification

### `summary.json` Structure

Top-level aggregates: `version`, `runId`, `generatedAt`, `totalRows`, `modes[]`, `scenarios[]`, `successRate`. Per-mode breakdowns with `successRate`, `count`, and `medians` for `wallMs`, `activeTokens`, `toolCalls`, `costUsd`. Per-scenario breakdowns with `count` and `successRate`.

### Session Transcripts (`sessions/`)

Full JSON transcripts of each agent session. Contains the prompt sent to the agent, every tool call and response, and the final completion. These are the primary source of truth -- all metrics in `results.csv` are derived from these transcripts.

**Note:** `sessions/pr-review-comment-001/baseline-iter-3.json` is absent because that iteration failed with a 120s timeout before producing a session transcript.

### Behavioral Analysis (`analysis/`)

Per-session analysis bundles generated by the agent-profiler's five built-in analyzers: Reasoning, Strategy, Efficiency, ToolPattern, and Error. Each file contains efficiency metrics (turns, backtracking), tool sequence patterns, strategy classification, and reasoning quality indicators.

**Note:** `analysis/pr-review-comment-001/baseline-iter-3-analysis.json` is absent (no session data to analyze).

## Verifying Results

To independently verify any number in the evaluation report:

1. Open `data/results.csv` and filter to the relevant mode/scenario/iteration
2. Cross-reference with the session transcript in `sessions/` for the raw tool call log
3. Check `data/results.json` for nested detail (individual tool call records, checkpoint verification responses)

To recompute aggregate statistics, use the per-iteration rows in `results.csv` and apply medians, bootstrap CIs, and permutation tests as described in [Interpreting Results](../../packages/eval/docs/methodology/interpreting-results.md).
